================================================================================
                    AGENT CHATBOT PROJECT DOCUMENTATION
                    Ollama + MongoDB + React + Node.js
================================================================================

PROJECT OVERVIEW
----------------
A fully local, free chatbot that answers questions about agents and login data
using Mistral 7B via Ollama. Everything runs locally with no cloud costs.

TECH STACK
----------
- Frontend: React + Vite
- Backend: Node.js + Express
- Database: MongoDB
- AI Model: Ollama (Mistral 7B)
- Cost: $0 (100% free and local)

================================================================================
                              PREREQUISITES
================================================================================

1. Node.js (v18+)
   Download: https://nodejs.org/

2. MongoDB (running locally on port 27017)
   Required Collections:
   - agents: UserName, Name, Nationality, Comp_Name, Date_establishment, 
             AgentID, CreatedDate, LastLogin
   - agentlogins: AGENTID, ID, LOGINDATE

3. Ollama
   Download: https://ollama.ai/
   Model: mistral:7b

================================================================================
                            INSTALLATION GUIDE
================================================================================

STEP 1: INSTALL OLLAMA & DOWNLOAD MISTRAL
------------------------------------------
# Install Ollama from https://ollama.ai/

# Download Mistral 7B model
ollama pull mistral:7b

# Verify it's running
ollama run mistral:7b

# Type /bye to exit


STEP 2: SETUP MONGODB
----------------------
# Start MongoDB (if not running)
mongod

# Verify connection
mongosh
use testdb
show collections


STEP 3: INSTALL BACKEND
------------------------
cd backend
npm install

# Configure .env file:
PORT=5000
MONGODB_URI=mongodb://127.0.0.1:27017/testdb
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=mistral:7b

# Start backend
npm start

# Expected output:
✅ Connected to MongoDB: mongodb://127.0.0.1:27017/testdb
✅ Loaded 12 intents
🚀 Server running on http://localhost:5000


STEP 4: INSTALL FRONTEND
-------------------------
cd frontend
npm install
npm run dev

# Open browser at:
http://localhost:3000

================================================================================
                          HOW THE SYSTEM WORKS
================================================================================

FLOW DIAGRAM:
-------------

User types: "hw many activ usrs r ther?"
        ↓
React Frontend sends to Node.js Backend
        ↓
Backend sends to Ollama: "What intent is this?"
        ↓
Ollama returns: "get_total_agents"
        ↓
Backend finds matching query in intents.json
        ↓
Backend executes MongoDB query
        ↓
Backend sends result to Ollama: "Make this user-friendly"
        ↓
Ollama returns: "There are 142 agents in the system!"
        ↓
React displays the formatted answer


DETAILED PROCESS:
-----------------

1. USER INPUT
   - User types question in React frontend
   - Frontend sends POST request to /api/chat

2. INTENT MATCHING
   - Backend receives question
   - Sends to Ollama with intent examples
   - Ollama identifies the matching intent

3. PARAMETER EXTRACTION
   - If intent needs parameters (email, date, etc.)
   - Ollama extracts them from user question

4. DATABASE QUERY
   - Backend finds query template in intents.json
   - Replaces parameters in query
   - Executes MongoDB query

5. RESPONSE FORMATTING
   - Raw data sent to Ollama
   - Ollama formats into natural language
   - Backend sends to frontend

6. DISPLAY
   - Frontend shows formatted response
   - If tabular data, displays in DataTable component

                          TROUBLESHOOTING

---------------------

ollama list

ollama serve

# Test model
ollama run mistral:7b


MONGODB CONNECTION ERROR
------------------------
# Check MongoDB is running
mongosh

# Verify database name
use testdb
show collections

# Check connection string in .env
MONGODB_URI=mongodb://127.0.0.1:27017/testdb


BACKEND ERRORS
--------------
# Check terminal logs for errors
# Verify .env file exists and has correct values
# Validate intents.json is valid JSON
# Check MongoDB collections exist


FRONTEND NOT CONNECTING
-----------------------
# Verify backend is running on port 5000
# Check VITE_API_URL in frontend/.env
# Check browser console for errors
# Verify CORS is enabled in backend


PORT ALREADY IN USE
-------------------
# Change PORT in backend/.env
# Update VITE_API_URL in frontend/.env accordingly
