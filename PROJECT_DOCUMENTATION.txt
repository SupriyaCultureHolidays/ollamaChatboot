================================================================================
                    🤖 AGENT CHATBOT - PROJECT DOCUMENTATION
================================================================================

OVERVIEW
--------
A fully local, free chatbot that answers questions about agents and login data 
using Mistral 7B via Ollama. No cloud dependencies, no API costs.

VERSION: 1.0
TECH STACK: React + Node.js + MongoDB + Ollama (Mistral 7B)
COST: $0 (100% free and local)

================================================================================
                          SYSTEM REQUIREMENTS
================================================================================

PREREQUISITES
-------------
✅ Node.js v18+ - https://nodejs.org/
✅ MongoDB running locally on port 27017
✅ Ollama installed - https://ollama.ai/
✅ Mistral 7B model downloaded: ollama pull mistral:7b

REQUIRED MONGODB COLLECTIONS
-----------------------------
1. agents - Fields: UserName, Name, Nationality, Comp_Name, Date_establishment, 
            AgentID, CreatedDate, LastLogin
2. agentlogins - Fields: AGENTID, ID, LOGINDATE

PORTS USED
----------
- Backend: 5000
- Frontend: 3000  
- MongoDB: 27017
- Ollama: 11434

================================================================================
                          INSTALLATION GUIDE
================================================================================

STEP 1: SETUP OLLAMA
--------------------
# Install Ollama from https://ollama.ai/
ollama pull mistral:7b
ollama run mistral:7b
# Type /bye to exit

STEP 2: SETUP MONGODB
---------------------
# Ensure MongoDB is running with your data
mongod
# Verify collections exist:
mongosh
use testdb
show collections

STEP 3: BACKEND SETUP
---------------------
cd backend
npm install
# Edit .env if needed:
PORT=5000
MONGODB_URI=mongodb://127.0.0.1:27017/testdb
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=mistral:7b

npm start
# Expected output:
# ✅ Connected to MongoDB: mongodb://127.0.0.1:27017/testdb
# ✅ Loaded 15 intents
# 🚀 Server running on http://localhost:5000

STEP 4: FRONTEND SETUP
----------------------
cd frontend
npm install
npm run dev
# Open http://localhost:3000

================================================================================
                          HOW THE SYSTEM WORKS
================================================================================

QUERY FLOW
----------
User types: "Show agents from India"
        ↓
React sends to Node.js backend (/api/chat)
        ↓
Backend extracts parameters (country: "India", page: 1, limit: 10)
        ↓
Backend sends to Ollama: "What intent is this?"
        ↓
Ollama returns: "get_agents_by_nationality"
        ↓
Backend finds matching query in intents.json
        ↓
Backend runs MongoDB query with pagination
        ↓
MongoDB returns: { data: [...], pagination: { page: 1, total: 47 } }
        ↓
Backend formats: "Showing 1-10 of 47 results."
        ↓
React displays answer + table with Prev/Next buttons

PAGINATION FLOW
---------------
User clicks [Next ▶] or types "show next"
        ↓
Backend reads session (no Ollama call needed!)
        ↓
Increments page: 1 → 2
        ↓
Runs same query with skip(10).limit(10)
        ↓
Returns page 2 results (< 100ms)

================================================================================
                          PERFORMANCE METRICS
================================================================================

QUERY TYPE              | TIME      | NOTES
------------------------|-----------|--------------------------------------
First query (Ollama)    | 2-3s      | Intent detection + DB query
Pagination (next/prev)  | < 100ms   | No Ollama, uses session
MongoDB find()          | < 50ms    | With indexes
MongoDB count()         | ~30ms     | For pagination total
Simple count query      | < 20ms    | No data retrieval

OPTIMIZATION TIPS:
- Add indexes on frequently queried fields
- Use pagination for large result sets
- Session storage avoids re-querying Ollama
- Limit Ollama tokens (num_predict: 50)

================================================================================
                          TROUBLESHOOTING
================================================================================

OLLAMA NOT RESPONDING
---------------------
# Check if Ollama is running
ollama list

# Restart Ollama service
ollama serve

# Verify model is available
ollama run mistral:7b

MONGODB CONNECTION ERROR
------------------------
# Check MongoDB is running
mongosh

# Verify database name
use testdb
show collections

# Check connection string in .env
MONGODB_URI=mongodb://127.0.0.1:27017/testdb

BACKEND ERRORS
--------------
# Check logs in terminal
# Make sure .env has correct values
# Verify intents.json is valid JSON
# Restart backend: npm start

INTENT NOT DETECTED CORRECTLY
-----------------------------
# Check console: "🎯 Detected intent: ..."
# If wrong, the Ollama prompt may need adjustment
# Add more examples to intents.json
# Restart backend after any changes

NO RESULTS RETURNED
-------------------
# Check console: "💾 DB Result: ..."
# Verify MongoDB has data
# Check query parameters: "📝 Parameters: ..."

PAGINATION NOT WORKING
----------------------
# Ensure intent has "pagination": true
# Check session storage in backend logs
# Verify frontend passes pagination prop

FRONTEND NOT LOADING
--------------------
# Check if backend is running on port 5000
# Verify CORS settings
# Check browser console for errors
# Ensure .env has correct API URL

================================================================================
                          ADDING NEW INTENTS
================================================================================

STEP 1: Edit backend/intents/intents.json
------------------------------------------
{
  "intent": "your_new_intent",
  "collection": "agents",
  "query": { "field": "value" },
  "pagination": true,           // Optional
  "defaultLimit": 10,           // Optional
  "sort": { "field": -1 },      // Optional
  "template": "Found {count} results."
}

STEP 2: Restart backend
-----------------------
cd backend
npm start

STEP 3: Test new intent
-----------------------
Ask a question that matches your new intent

TEMPLATE VARIABLES:
- {param} - Extracted parameter
- {count} - Result count
- {field} - Any field from result object
- {from}, {to}, {total} - Pagination metadata

EXAMPLE NEW INTENT:
{
  "intent": "get_agents_by_status",
  "collection": "agents",
  "query": { "Status": "{status}" },
  "pagination": true,
  "defaultLimit": 10,
  "template": "Found {count} {status} agents."
}
